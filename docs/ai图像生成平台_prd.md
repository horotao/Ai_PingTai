# AI 图像生成工作台 PRD（纯前端版）

## 1. 文档信息

- **文档名称**：AI 图像生成工作台产品需求文档
- **文档版本**：V0.3 纯前端精简版
- **产品阶段**：MVP 规划阶段
- **适用范围**：Web 端
- **技术栈**：纯前端（Next.js / Vite + React + shadcn/ui + Tailwind CSS）
- **后端依赖**：无。所有数据存储在用户本地浏览器。

---

## 2. 项目概述

### 2.1 项目背景
当前 AI 图像生成服务分散在不同平台，用户频繁切换工具、重复配置参数。本项目提供一个**本地化、零账号、自带 Key** 的图像创作工作台：用户填入自己的 Apimart API Key 即可使用，所有配置与记录存储在浏览器本地，图片由用户自行下载保管，平台本身不持有任何用户数据。

### 2.2 项目目标
1. 提供简洁统一的 Apimart 图像生成接入界面。
2. 用户使用零门槛：无需注册登录，打开即用。
3. 所有敏感数据（API Key、历史记录）仅存储在用户本地浏览器。
4. 图片由用户自行下载保存，平台不做云端存储。
5. MVP 闭环：填入 Key → 在首页输入创作 → 查看结果 → 下载保存。
6. 首页即工作台：输入、参数、提交、结果展示统一在一个主页面内完成，无需页面跳转。

### 2.3 产品定位
面向创作者的 **纯前端 AI 图像生成工具**。形态上是一个"本地工作台"，不是平台也不是社区。强调隐私自持、轻量、即开即用。

### 2.4 本期接入范围
MVP 阶段接入 Apimart 图像生成接口的两个模型：

1. **`gpt-image-2`**：异步图像生成，文生图 / 图生图，13 种比例，单次 1 张。
2. **`gpt-image-2-official`**：OpenAI 官方渠道，异步 `task_id`，文生图 / 图生图 / 局部重绘，`1k / 2k / 4k` 分辨率，单次最多 4 张。

## 3. 用户与场景

### 3.1 目标用户
- 设计师、电商视觉从业者
- 内容创作者、自媒体运营
- 已持有 Apimart API Key 的 AI 重度用户
- 注重数据隐私、不希望账号信息被第三方平台持有的用户

### 3.2 核心使用场景
1. 用户首次打开页面，进入 API 配置页填写自己的 Apimart Key。
2. Key 保存在浏览器本地后，进入工作台。
3. 输入提示词 / 上传参考图 / 选择模型与参数。
4. 提交任务，前端轮询 Apimart 任务状态。
5. 任务完成后查看结果，点击下载保存到本地。
6. 在历史记录页查看过往任务（仅本浏览器内可见）。

## 4. 产品原则

1. **零账号**：不做注册登录，不收集任何用户信息。
2. **本地优先**：所有配置与记录存于用户浏览器，清除浏览器数据即全部清空。
3. **创作入口优先**：首页即输入框，不做宣传页。
4. **异步清晰**：任务状态（提交 / 处理中 / 完成 / 失败）反馈明确。
5. **用户自持数据**：生成的图片由用户主动下载保存，平台不中转、不留存。

## 7. 功能需求

### 7.1 主页面（首页即工作台）

#### 功能目标
作为产品唯一的核心页面，承担从输入、参数配置到结果展示的**全流程创作闭环**。无需跳转任何"工作台"子页面，用户在当前页就能完成所有创作操作。

#### 设计参考
参考即梦、Seedance 等产品的形态：底部一个大输入框统一承载所有输入动作（文本 + 上传 + 模式切换 + 提交），详细参数通过"参数抽屉"按需展开，中心区展示引导文案或当前任务结果，形成"聊天式创作"体感。

#### 页面布局

```
┌──────────────────────────────────────────────────────────┐
│ Logo │ 会话标题 ▾             搜索 筛选器(时间/类型等)  │ ← 顶栏
├──────┬───────────────────────────────────────────────────┤
│ 📁   │                                                   │
│ 历史 │                                                   │
│      │         中心结果区 / 引导视觉区                   │
│ ⚙️   │         (未生成时：引导文案 + 视觉符号)           │
│ 设置 │         (已生成：结果图 / 进行中任务状态)         │
│      │                                                   │
│      │  ┌─────────────────────────────────────────────┐  │
│      │  │ 📎  提示词输入框...                       ↑ │  │ ← 底部主输入区
│      │  │ [模型▾] [参数] [比例▾] [张数] ...           │  │
│      │  └─────────────────────────────────────────────┘  │
└──────┴───────────────────────────────────────────────────┘
```

#### 7.1.1 顶栏
- **左侧**：产品 Logo + 当前会话标题（可选，例如"未命名对话"，本地生成 UUID）。
- **右侧**：搜索图标、筛选器（时间、生成类型、操作类型），用于中心结果区的历史滚动筛选。
- 右上角不展示用户信息（无账号系统）。

#### 7.1.2 左侧边栏
窄边栏，仅图标，悬停显示 tooltip：
- 📁 **历史记录** — 进入历史页
- ⚙️ **设置** — 进入 API 配置页

> MVP 只需这两个图标。后续扩展（画布、资产、灵感）作为 P2。

#### 7.1.3 中心结果区
页面主区域，根据状态呈现不同内容：

| 状态 | 表现 |
|---|---|
| 初次进入（无任务） | 留白 + 轻量视觉符号 + 一句文案"开启你的新创作" |
| 任务进行中 | 骨架屏 / Loading 动画 + 当前任务状态文案 |
| 任务完成 | 结果图网格（支持点击大图查看） |
| 连续创作 | 以"对话式"向上滚动累积历史结果（参考 ChatGPT 交互） |

#### 7.1.4 底部主输入区（核心模块）

**基础层（始终可见）**
- 📎 参考图上传按钮（点击选择文件 / 支持拖拽进入整个输入区）
- 多行提示词输入框（自动高度，支持 `Shift+Enter` 换行，`Enter` 提交）
- 已上传参考图缩略图（悬浮在输入框上方，可删除）
- 🚀 提交按钮（右下角）

**快捷栏（输入框下方一行）**
- **模型选择** — 下拉：`gpt-image-2` / `gpt-image-2-official`（默认选中推荐模型）
- **比例** — 常用比例快捷选择（例如 `1:1 / 16:9 / 9:16`），其余比例在参数抽屉中
- **张数**（`n`）— 仅 `gpt-image-2-official` 可调，默认 1
- **参数** — 齿轮图标，点击展开完整参数抽屉

**参数抽屉（点击"参数"按钮后从底部上拉展开，或从右侧滑入）**
承载所有次要参数，避免默认状态下视觉噪音：

| 字段 | `gpt-image-2` | `gpt-image-2-official` |
|---|---|---|
| `size` | 13 种比例（完整） | 13 种比例（4K 仅 6 种） |
| `resolution` | — | `1k / 2k / 4k` |
| `quality` | — | `auto / low / medium / high` |
| `background` | — | `auto / opaque` |
| `moderation` | — | `auto / low` |
| `output_format` | — | `png / jpeg / webp` |
| `output_compression` | — | `0-100`（仅 jpeg/webp） |
| `n` | 固定 1 | `1-4` |
| `image_urls` | 最多 16 张 | 最多 16 张 |
| `mask_url` | — | 局部重绘用 |
| `official_fallback` | 布尔开关 | — |

切换模型时参数抽屉内字段动态联动，不支持的字段不展示。

#### 7.1.5 输入区交互要点
- 草稿自动保存到 `localStorage`，刷新不丢失。
- 未配置 API Key 时，输入框展示占位："请先在设置中填入 Apimart API Key"，提交按钮禁用，点击禁用状态会弹出跳转设置的气泡。
- 参考图上传最多 16 张，与 `image_urls` 约束一致。
- 拖拽文件到窗口任意位置均可触发上传（整个主区变为 drop zone）。

#### 7.1.6 参数校验（前端）
- `gpt-image-2`：`n` 只能为 1；`size` 只能传比例值（非像素字符串）。
- `gpt-image-2-official`：4K 仅支持 `16:9 / 9:16 / 2:1 / 1:2 / 21:9 / 9:21`，选择 4K 时 `size` 下拉自动收窄。
- 使用 `mask_url` 时必须至少存在 1 张参考图。
- 仅当 `output_format` 为 `jpeg / webp` 时允许填 `output_compression`。

#### 7.1.7 任务状态
提交后，结果区立即插入一条"进行中"任务卡片，轮询过程中状态文案动态更新：
- `submitted / pending` → 已提交
- `in_progress / processing` → 处理中
- `completed` → 已完成（替换为结果图）
- `failed` → 已失败（显示错误原因 + 重试按钮）

#### 7.1.8 结果交互
每个结果图支持：
- 点击查看大图
- 复制结果 URL
- **下载到本地**（核心动作，单张下载 / 批量下载 ZIP）
- 一键"使用此提示词再次创作"（回填到输入框）
- 一键"以此图为参考图"（自动带入 `image_urls`，需先解决 §9.2 的 URL 问题）

#### 核心流程
1. 用户打开页面，若无 Key 则引导进设置。
2. 在底部输入框输入提示词、上传参考图、选择模型。
3. 如需精细调参，点击"参数"展开抽屉。
4. 提交 → 结果区插入进行中任务卡片 → 本地写入 IndexedDB。
5. 前端轮询 Apimart 任务状态（§9.3）。
6. 完成后结果图在当前页直接展示，用户下载保存。
7. 可直接在当前页继续下一次创作，形成对话式连续创作。

#### 功能目标
查看本浏览器内的历史任务记录。

#### 功能说明

- 清除浏览器数据 / 切换浏览器 / 切换设备后记录即消失。
- 页面顶部明确提示："记录仅保存在本浏览器中，建议及时下载所需图片"。

- 模型名称

- 操作：查看 / 删除 / 再次创作 / 下载

#### 功能要求
- 按时间倒序展示
- 支持分页或虚拟滚动
- 支持批量删除 / 清空全部
- 支持导出 JSON（方便用户备份）

> **注意**：Apimart 返回的图片 URL 通常有时效性。历史记录中仅保存 URL，一旦过期无法再次访问。必须在 UI 上明确提示用户及时下载。

#### 页面字段

- API Key（输入框，默认隐藏显示，提供👁切换）

## 8. 本地数据设计

所有数据存储在用户浏览器本地，不使用任何云端数据库。

### 8.1 localStorage
轻量配置，同步访问：

| Key | 内容 |
|---|---|
| `apimart_configs` | Apimart 配置数组（含 Key、默认模型、启用状态） |
| `draft_prompt` | 未提交的提示词草稿 |
| `ui_preferences` | 界面偏好（主题、默认模型等） |

### 8.2 IndexedDB
较大数据，异步访问：

**`tasks` 表**
- `id`（本地生成 UUID）
- `apimart_task_id`
- `model_name`
- `task_type`（文生图 / 图生图 / 局部重绘）
- `prompt`
- `params`（JSON）
- `input_file_refs`（参考图的本地引用或外链 URL）
- `status`
- `result_urls`（Apimart 返回的图片 URL 数组）
- `created_at`
- `updated_at`
- `error_message`

**`uploads` 表（若采用转外链方案）**
- `id`
- `file_name`
- `local_blob`（可选，存 Blob 用于预览）
- `remote_url`（用户填入的外链 URL）
- `file_type`
- `file_size`
- `created_at`

### 8.3 导入 / 导出
- 提供"导出全部数据"按钮：导出 JSON 文件（含配置 + 历史）。
- 提供"导入数据"按钮：从 JSON 恢复。
- 这是用户在不同浏览器 / 设备之间"同步"的唯一方式。

## 9. 关键技术问题与对策

> 这几个问题是纯前端架构下的硬约束，必须在开发前明确方案。

### 9.1 CORS 跨域问题
浏览器直接请求 Apimart 接口可能被 CORS 拦截。需先确认：

- ✅ 若 Apimart **已开放 CORS**（响应头含 `Access-Control-Allow-Origin: *` 或允许你的域名）：可纯前端直连。
- ❌ 若 Apimart **未开放 CORS**：需要一个极简代理。推荐以下方案（按成本排序）：
  1. 部署在 Vercel / Cloudflare Workers 的无状态代理函数（单文件，不涉及数据库）。
  2. 使用浏览器扩展绕过（仅开发调试用，不推荐生产）。
  3. 用户自建本地反向代理（不现实，放弃）。

> **建议**：开发前先用 `curl` + `Origin` 头测试 Apimart 的 CORS 响应，确定方案。即使需要代理，该代理也只做请求透传，不落盘、不持有 Key，不破坏"纯前端"属性。

### 9.2 图生图的参考图 URL 问题
Apimart 要求 `image_urls` 是**公网可访问的 URL**，而纯前端无法提供图片托管。三种方案：

| 方案 | 描述 | 优缺点 |
|---|---|---|
| A. 用户自带图床 | 在上传区让用户填入已经存在于公网的图片 URL | ✅ 实现简单，零依赖<br>❌ 用户体验差 |
| B. 接入第三方图床 API | 集成 SM.MS / ImgBB 等免费图床，由用户填写图床 Key | ✅ 体验较好<br>❌ 多一层依赖与 Key 管理 |
| C. Base64 内联 | 若 Apimart 支持 `data:` URL 或 base64 字段，直接内联 | ✅ 最理想，完全本地化<br>❌ 取决于 Apimart 是否支持（需查 API 文档） |

> **建议顺序**：先确认 C 是否可行 → 否则采用 A 作为 MVP → 后续迭代 B。

### 9.3 异步任务轮询策略
纯前端轮询需注意：
- 页面关闭即丢失轮询上下文。重新打开时需从 IndexedDB 恢复"进行中"任务并继续轮询。
- 轮询间隔建议指数退避：前 30 秒 2s/次，之后 5s/次，超过 5 分钟改为 10s/次。
- 2K / 4K 任务可能耗时数分钟，需在 UI 上明确提示"此任务可能较慢，可切换到其他页面"。
- 同时进行多个任务时需做并发控制，避免过多请求。

### 9.4 图片下载
- Apimart 返回的是远程 URL，直接 `<a download>` 在跨域情况下可能失效。
- 需用 `fetch` 拿到 Blob → `URL.createObjectURL` → 触发下载。
- 这一步同样可能触发 CORS，需与 §9.1 合并考虑。

### 9.5 API Key 安全
- 存在 `localStorage` 的 Key 任何同源脚本均可读取。
- 严格控制第三方脚本引入（避免任何来历不明的 `<script>`）。
- 可选加固：用户设置"本地密码"，通过 Web Crypto API 派生密钥后加密 API Key，每次使用输入密码解密。

## 10. 技术实现原则

### 10.1 前端框架
- Next.js（若需要 SEO 与 SSR）或 Vite + React（纯 SPA，部署更简单）。
- 鉴于本项目**完全不需要服务端渲染**，**推荐 Vite + React**，部署到 Cloudflare Pages / Vercel / Netlify 静态托管均可。
- UI 库：shadcn/ui + Tailwind CSS。
- 状态管理：Zustand 或 React Context（数据量小即可）。
- 本地存储封装：`idb` 库（IndexedDB 的 Promise 封装）。

### 10.2 不使用的东西
- ❌ 数据库（PostgreSQL / Supabase）
- ❌ 用户认证系统
- ❌ 云端文件存储
- ❌ 服务端 API（可选代理除外）
- ❌ 后台管理

### 10.3 部署
- 纯静态托管。一个 `dist/` 目录即可。
- 无环境变量、无密钥，所有配置由用户在浏览器内填写。

## 11. 非功能需求

### 11.1 性能
- 首次加载 < 2s。
- 参数切换即时响应。
- 大量历史记录时使用虚拟滚动，避免卡顿。

### 11.2 可用性
- 无 Key 时主输入区禁用并明确提示。
- 失败状态必须展示 Apimart 返回的错误信息原文 + 可读解释。
- 4K 高质量等慢任务需明确标注预期耗时。

### 11.3 安全性
- API Key 不发送到除 Apimart 外的任何服务器。
- 文档和 UI 中明确告知存储位置与隐私策略。
- 可选本地加密。

### 11.4 兼容性
- 优先支持桌面端 Chrome / Edge / Firefox / Safari 最新两个版本。

- IndexedDB 与 Web Crypto 在隐私模式下可能受限，需做降级提示。

## 13. MVP 验收标准

### 13.1 首次使用
- 首次打开页面即可浏览，无需任何账号。
- 未配置 Key 时主输入区禁用并清晰引导跳转设置。

### 13.2 API 配置
- 可新增 / 编辑 / 删除 Apimart 配置。
- 可设置默认启用配置。
- 可测试 API Key 有效性。
- Key 仅存储在 `localStorage`，刷新后仍然保留。

### 13.3 主页面（工作台）
- 所有创作动作均在主页面内完成，无需跳转子页面。
- 底部主输入区包含：提示词、参考图上传、模型选择、比例、参数抽屉、提交。
- 可选择两个模型之一，参数抽屉内字段随模型动态变化。
- 默认参数（模型、比例）在输入框下方快捷栏直接可见。
- 次要参数（`resolution`、`quality`、`background` 等）在参数抽屉内。
- 可提交任务并在中心结果区看到进行中状态。
- 轮询状态实时更新，完成后结果图直接呈现在主页面。
- 结果图可查看大图、下载、复制 URL、一键再次创作。
- 提交新任务后，历史结果以对话式向上累积，不跳转页面。

### 13.4 历史记录
- 任务记录自动写入 IndexedDB。
- 历史记录页可查看、删除、再次使用。
- 可导出 / 导入 JSON。
- 页面明确提示"记录仅在本浏览器"。

### 13.5 跨设备
- 在浏览器 A 填入 Key 并创作，浏览器 B 看不到任何数据（符合预期）。
- 通过导出 JSON 可在 B 导入恢复。

## 15. 风险与待确认项

### 15.1 待确认（开发前必须明确）
1. **Apimart 是否支持浏览器直连（CORS 策略）？** 决定是否需要代理。
2. **Apimart 的 `image_urls` 字段是否支持 base64 / data URL？** 决定图生图方案。
3. **Apimart 返回的结果图 URL 有效期多长？** 决定历史记录的可用性与是否需要自动转存。
4. **Apimart 是否提供 Key 余额 / 配额查询接口？** 决定配置页能否展示额度信息。

### 15.2 已知风险
1. 用户清除浏览器数据 → 历史全部丢失。需在 UI 上反复强调"及时下载 + 导出备份"。
2. Apimart 图片 URL 过期 → 历史记录中旧图可能无法加载。可在 P2 阶段做自动转存 Blob。
3. API Key 存储在 localStorage 存在被恶意脚本窃取的风险。本地加密方案列入 P2。
4. 2K / 4K 任务耗时长，用户可能关闭页面导致任务"丢失"。需做页面恢复机制（§9.3）。
5. 纯前端无法做请求限流、审计、滥用防护。这是 MVP 的有意权衡。

## 16. 与原方案（有后端版）的主要差异

| 维度 | 原方案（V0.2） | 本方案（V0.3） |
|---|---|---|
| 用户系统 | Supabase Auth | 无 |
| 数据存储 | Postgres + RLS | localStorage + IndexedDB |
| 文件存储 | Supabase Storage | 无（用户本地下载） |
| API 请求 | Edge Functions 代理 | 浏览器直连 / 极简代理 |
| Key 安全 | 服务端加密存储 | localStorage（可选本地加密） |
| 跨设备同步 | 天然支持 | 靠 JSON 导入导出 |
| 部署成本 | 中（Supabase + 前端） | 低（静态托管即可） |
| 隐私性 | 数据在 Supabase | 数据在用户浏览器 |
| 适合规模 | 多用户产品 | 个人工具 / 内部工具 |
